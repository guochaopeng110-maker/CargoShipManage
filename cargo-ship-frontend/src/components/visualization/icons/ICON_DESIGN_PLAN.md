# 图标设计方案文档

## 文档概述

本文档定义了可视化组件系统第一期需要实现的监测点图标清单，基于 `monitoring_and_alarm_definitions.md` 中的监测点定义，按照重要性和使用频率进行优先级排序。

**创建日期**：2025-12-12
**版本**：v1.0
**目标**：第一期实现 30-40 个关键图标

---

## 优先级分类标准

### P0 - 关键图标（必须实现）
- **标准**：驾控台页面展示的核心监测点
- **特征**：数值类型、标记为"关键指标"、影响船舶安全和动力
- **数量目标**：15-20 个

### P1 - 重要图标（优先实现）
- **标准**：各监控页面的主要监测点
- **特征**：标记为"关键指标"、用于状态判断
- **数量目标**：15-20 个

### P2 - 一般图标（后续实现）
- **标准**：辅助监测点、非关键指标
- **特征**：补充信息、趋势监测
- **数量目标**：暂不实现，使用占位符图标

## 图标设计规范

### 视觉风格
- **风格**：线性图标（Line Icons），2px 线宽
- **尺寸**：24x24px 基准，支持 16px、32px、48px、64px 缩放
- **颜色**：单色设计，通过组件动态着色
  - 正常状态：`#3b82f6`（蓝色）
  - 警告状态：`#f59e0b`（黄色）
  - 严重状态：`#ef4444`（红色）
- **圆角**：统一使用 2px 圆角
- **留白**：图标边缘留 2px 内边距

### SVG 优化要求
1. 移除所有 metadata、注释、编辑器特定属性
2. 合并路径，减少节点数量
3. 使用相对路径（小写命令）减小文件大小
4. 移除不必要的 group 标签
5. 统一使用 `currentColor` 以支持动态着色

### 语义化要求
1. 图标必须在无颜色情况下依然可识别
2. 关键特征明显，16px 尺寸下仍可辨认
3. 避免过于复杂的细节
4. 同类型图标保持视觉一致性（如所有温度图标都使用温度计）

## 图标命名规范

SVG 文件命名格式：`{系统缩写}-{监测点类型}.svg`

示例：
- `bat-total-voltage.svg` - 电池总电压
- `bat-soc.svg` - 电池SOC
- `motor-speed.svg` - 电机转速
- `inverter-input-voltage.svg` - 逆变器输入电压
- `busbar-voltage.svg` - 母排电压

| `boolean` | `default-status.svg` | 通用状态图标（圆形 + 开关符号） |
