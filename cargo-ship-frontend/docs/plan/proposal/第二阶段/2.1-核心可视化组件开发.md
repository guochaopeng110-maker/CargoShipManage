这个阶段的目标是开发出驱动新版界面的核心引擎，包括前端的可视化组件和后端的实时数据动脉。

  我将首先提出此阶段第一个核心任务的概念草稿，它聚焦于数据可视化。

  ---

  概念草稿：[阶段二] 任务2.1 - 开发核心可视化组件库

   * 目标 (Objective):
      
  创建一套具有强动画效果、高信息密度且可复用的核心可视化组件。这套组件将作为所有监控类页面的视觉基石，
  严格遵循重构方案中“丰富且直观的数据可视化”的核心原则，将枯燥的数据转化为富有生命力的图形信息。

   * 核心概念 (Core Concept):
       1. 指标专属图标体系 (Metric-Specific Icon System):
           * 核心任务: 必须创建一个 Icon 组件库或图标映射规则。
           * 严格要求: 摒弃“按类型（如电压、温度）分类”的通用图标做法。我们必须为
             monitoring_and_alarm_definitions.md 中定义的每一个监测点ID (`monitoringPointId`)
             设计并匹配一个独特的、具有业务意义的SVG图标。例如，total_voltage 和 cell_voltage_max
             应有各自不同的专属图标。
           * 实现方式: 该组件接收一个 monitoringPointId 作为 prop，并渲染对应的SVG图标。
       2. 动画优先原则 (Animation-First Principle):
           * 核心任务:
             为所有图表和关键状态指示器（尤其是在监测点图标上）融入流畅且视觉冲击力强的动画效果。
           * 技术选型: 优先考虑使用 framer-motion 或其他高级CSS动画技术来实现。
           * 严格要求:
             动画必须作为组件的核心功能来设计，而非事后添加。例如，仪表盘指针应以动画形式平滑过渡；设
             备状态从“正常”变为“告警”时，其图标必须有显著的脉冲、闪烁或颜色渐变动画，以强制吸引用户注
             意力。
       3. 关键组件开发 (Key Component Development):
           * `GaugeChart.tsx`: 用于展示百分比或范围值的动态仪表盘。
           * `DonutChart.tsx`: 用于展示简单分布的环形图。
           * `MetricCard.tsx` / `StatusIndicator.tsx`:
             一个高度复用的“指标卡片”组件，它将整合指标专属图标、实时数值和基于状态的动画，将成为构成“
             监控墙”的基本单元。
       4. 响应式与实时性 (Responsive & Real-time Ready):
           * 所有组件都必须是响应式的，能够在不同尺寸的屏幕和布局容器中正常显示。
           * 组件的接口设计必须能够轻松接收来自 Zustand stores
             的实时数据流，并高效地触发重渲染和动画。

   * 预期成果 (Expected Outcome):
       * 一个包含所有监测点专属SVG图标的图标库或映射表被建立。
       * 一套核心可视化组件（如 GaugeChart, MetricCard 等）被开发完成。
       * 这些组件具备了重构方案所要求的、以动画为核心的交互特性。
       * 可以通过 Storybook
         或一个临时的测试页面，清晰地展示这些组件在接收到模拟数据变化时，能够流畅地执行出对应的动画效
         果，验证其已准备好与真实数据对接。
