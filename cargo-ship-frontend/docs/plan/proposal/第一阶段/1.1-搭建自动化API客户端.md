  概念草稿：[阶段一] 任务1.1 - 搭建自动化API客户端                                                    
                                                                                                      
   * 目标 (Objective):                                                                                
                                                                                                      
  从根本上提升前端项目的类型安全、可维护性和开发效率，为后续所有功能的重构奠定坚实、可靠的数据基础。  
                                                                                                      
   * 核心概念 (Core Concept):
       1. 引入代码生成工具: 选用一个成熟的OpenAPI客户端生成工具（首选
          openapi-typescript-codegen），并将其集成到项目的开发依赖和脚本中。
       2. 单一信源: 将 docs/data/http-api.json (OpenAPI v3 规范文件)
          作为生成API客户端的唯一、不可动摇的真实来源 (Single Source of Truth)。
       3. 自动化生成: 该工具应能自动生成所有与后端HTTP API调用相关的 TypeScript类型定义 和
          请求函数/方法。
       4. 代码隔离: 生成的客户端代码应放置在一个独立的、清晰标识的目录中（例如 src/services/api 或
          src/services/generated），并被.gitignore忽略（因为它是构建产物），以避免与手写代码混淆。
       5. 废弃旧代码: 新的自动化工作流将完全取代现有的、手写的服务文件（如alarms-service.ts,
          auth-service.ts, equipment-service.ts 等）。

   * 预期成果 (Expected Outcome):
       * 一个完全类型安全的、与后端API定义严格一致的TypeScript API客户端。
       * 一个npm script (例如 npm run generate:api)，开发者可以通过运行它来随时根据 http-api.json     
         的更新重新生成客户端。
       * 为后续的 stores 和 components 重构提供稳定、无需mock、类型安全的数据调用方法。
       * 旧的手写API服务被彻底移除的计划。
