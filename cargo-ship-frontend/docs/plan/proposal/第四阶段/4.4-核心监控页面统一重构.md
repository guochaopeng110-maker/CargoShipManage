主题: 应用“监控墙”模板统一重构核心监控页面

  背景:
  根据前端重构方案第四阶段计划，我们需要将“监控墙”模板模式应用到一系列核心监控页面。为了保持一致性并优化开发流程，我们将对 PropulsionMonitoringPage
  (推进系统)、InverterMonitoringPage (逆变器)、PowerDistributionPage (配电系统) 和 AuxiliarySystemsPage (辅助系统) 进行统一重构。

  目标:
  将上述四个监控页面全部重构为符合“监控墙”模式的页面。每个页面将专注于展示各自系统下所有关键监测点指标，并全面集成动态图标、动画效果及统一的数据驱动机制，同时 
  设有独立的告警信息区域。

  核心思想:
   1. 统一模板应用: 所有四个监控页面都将完全参照 BatteryMonitoringPage 的“监控墙”布局和设计模式进行重构。
   2. 系统指标集中展示: 每个页面的顶部区域将集中展示其对应系统（推进、逆变器、配电、辅助）的所有关键监测点指标，确保信息的全面性和直观性。
   3. 强化视觉效果: 为每个系统内的所有监测点指标设计并匹配独特的 SVG
      动态图标。这些图标及相关数值必须集成流畅且具有视觉冲击力的动画效果，以实时反映数据变化、告警状态和系统健康度。
   4. 一致数据驱动: 所有页面的数据流（包括实时更新）将完全通过 realtime-service 和对应的 monitoring-store
      驱动。确保数据获取、处理和展示逻辑在所有监控页面中保持一致和高效。
   5. 独立告警区域: 每个页面的底部区域将专门用于展示与当前系统相关的告警信息，提供清晰的系统级告警视图。
   6. 最大限度组件复用:
      在重构过程中，将最大限度地复用在第二、三阶段已开发的核心可视化组件、图标系统和数据管理逻辑，以确保代码质量、开发效率和用户体验的一致性。

  预期交付物 (高层次):
   * PropulsionMonitoringPage、InverterMonitoringPage、PowerDistributionPage 和 AuxiliarySystemsPage 的统一页面结构和布局。
   * 各系统所有关键监测点指标的动态、可视化“监控墙”展示。
   * 为各系统指标定制的动态 SVG 图标和动画效果。
   * 与 realtime-service 和 monitoring-store 深度集成的实时数据流。
   * 各页面底部独立的系统告警信息展示区。
