概念草稿：[阶段三] 任务3.2 - 确立 `HealthAssessmentPage` 页面模板

   * 目标 (Objective):

  设计并确立一个标准化的、以“结论”为导向的页面模板。此页面的核心目的不是展示原始的实时数据，而是呈现基
  于数据分析得出的健康状态、诊断建议和趋势预测，为管理者提供一个高级、可执行的全局健康概览。

   * 核心概念 (Core Concept):
       1. “总-分-细”三段式布局 (Total-Split-Detail Layout):
           * 顶部 (总) - 全船健康仪表盘 (Overall Health Scorecard):
               * 组件: 使用一个尺寸巨大、动画效果突出的 GaugeChart 或类似的“计分卡”组件。
               * 内容:
                 展示整船的综合健康评分，并根据健康等级（如“良好”、“一般”、“较差”）动态改变背景色或主
                 题色，同时清晰标示出等级文字。
           * 中部 (分) - 系统健康卡片矩阵 (System Health Grid):
               * 布局:
                 采用响应式网格布局，为每一个核心设备系统（如推进、电池、逆变器等）展示一个独立的“健康
                 卡片”。
               * 组件: 每个健康卡片 (SystemHealthCard.tsx) 都是一个微型仪表盘，至少包含：
                   * 该分系统的独立健康评分。
                   * 一个迷你的、内嵌的趋势图（“Sparkline”），可视化展示该系统近期健康分的变化。
                   * 该系统当前的活跃告警（高危/紧急）数量。
                   * 交互: 整个卡片可点击，并直接导航至对应系统的详细监控页面（如
                     BatteryMonitoringPage）。
           * 底部 (细) - 历史报告列表 (Historical Reports List):
               * 组件: 使用一个带分页功能的数据表格 (DataTable)。
               * 内容:
                 列出所有历史生成的健康评估报告。表格应包含报告ID、生成日期、涉及设备、健康分数以及查
                 看详情的操作入口。
       2. 混合数据流模式 (Hybrid Data Flow):
           * 实时健康数据: 顶部的“全船健康仪表盘”和中部的“系统健康卡片矩阵”的数据，主要来源于
             health-store。该 store 的数据由 realtime-service 推送的 equipment:health:update
             等事件实时更新。
           * 历史报告数据: 底部的“历史报告列表”数据，则通过 health-store 中的一个 action 来触发，该
             action 调用 api-client 从 /api/reports/health 接口按需获取分页数据。
           * 跨`store`数据聚合: “系统健康卡片”上显示的“活跃告警数”则来源于
             alarms-store，这展示了模板页面如何聚合多个 store 的数据来构建一个全面的视图。

   * 预期成果 (Expected Outcome):
       * 一个清晰的、可复用的“总-分-细”三段式页面布局模板被确立。
       * 一个新的、可复用的 SystemHealthCard.tsx 组件被创建。
       * HealthAssessmentPage 成为一个功能完善的高级概览页面，它成功地演示了一种混合数据流模式：通过  
         WebSocket/stores 实现实时分数更新，同时通过 API/stores 实现历史列表的按需加载。
       * 该页面有效地传达了设备健康状态，并为用户深入研究问题提供了清晰的导航路径。

  * 注意：由于后端更改了一个关于健康评估的接口: /api/reports/health, 并且已经执行了npm run generate:api命令，所以GenerateHealthReportDto.ts和HealthReport.ts,以及Service.ts有变更，必须使用中文回复。
