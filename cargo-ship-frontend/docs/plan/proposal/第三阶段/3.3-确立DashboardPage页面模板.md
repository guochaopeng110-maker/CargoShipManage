概念草稿：[阶段三] 任务3.3 - 确立 `DashboardPage` (驾控台) 页面模板

   * 目标 (Objective):

  为应用的“着陆页”——DashboardPage——建立一个模板。此页面必须是一个高信息密度的“指挥中心”，将来自不同系 
  统的最关键实时信息聚合到单一视图中，使用户能够“一目了然”地快速评估全局态势。

   * 核心概念 (Core Concept):
       1. 模块化小组件布局 (Modular Widget-Based Layout):
          页面主体由一个可定制的、响应式的网格构成，网格中填充着各种功能独立的“小组件” (Widget)。
           * 核心组件1 - 关键指标墙 (Critical Metrics Wall):
               * 定位: 跨系统的“超级仪表盘”，展示整船的生命体征。
               * 内容: 精心挑选 8-12
                 个全船最核心、最关键的监测点指标（如：电池SOC、主推电机转速、直流母线电压等）。
               * 组件: 使用 MetricCard
                 组件矩阵，每个卡片都展示其指标的专属动画图标、实时数值和运行状态。
           * 核心组件2 - 告警摘要 (Alarm Summary):
               * 定位: 快速掌握当前的告警态势。
               * 内容:
                   * 提供按告警等级（critical, high, medium, low）分类的实时统计数量。
                   * 一个滚动条或短列表，循环展示最新发生的 2-3 条紧急告警的摘要信息。
                   * 一个显眼的、可直接跳转至 Alarm Center（告警中心）的快捷入口。
           * 核心组件3 - 健康速览 (Health Quick-View):
               * 定位: 整船健康状况的紧凑型总结。
               * 内容: 一个迷你版的 HealthAssessmentPage 顶部的健康仪表盘
                 (GaugeChart)，显示总健康分和等级。
               * 交互: 整个组件可点击，作为快速进入完整 HealthAssessmentPage（健康评估页）的快捷方式。
           * 核心组件4 - 快捷操作 (Quick Actions):
               * 定位: 提供高频功能的一键直达入口。
               * 内容: 一个简洁的面板，包含清晰的图标和文字链接，指向如
                 DataQueryPage（数据查询）、DeviceManagementPage（设备管理）等常用页面。
       2. “超级消费者”数据流 (Super-Consumer Data Flow):
           * DashboardPage 是一个纯粹的“状态消费者”，它将同时从多个 stores 中获取数据来填充自己。
           * “关键指标墙”的数据来源于 monitoring-store。
           * “告警摘要”的数据来源于 alarms-store。
           * “健康速览”的数据来源于 health-store。
           * 页面本身不执行任何数据请求逻辑，完全依赖于已注入到 stores 中的实时数据，实现“被动”刷新。

   * 预期成果 (Expected Outcome):
       * 一个模块化的、基于小组件的 DashboardPage 页面模板被确立。
       * 该页面成功演示了如何通过组合多个独立的 Zustand stores 来构建一个复杂、高度聚合的视图。       
       * DashboardPage
         成为一个功能强大且视觉吸引力强的应用入口，为用户提供即时、可执行的全局状态洞察。
       * 可能会产出一些可复用的小组件（如 AlarmSummaryWidget,
         HealthQuickViewWidget），供项目其他地方使用。
