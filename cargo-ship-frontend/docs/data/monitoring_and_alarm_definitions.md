## 3. 告警信息定义

### 3.1 告警等级

系统采用四级告警等级：

- **critical (紧急)**: 需要立即处理的严重告警，可能导致设备损坏或系统停机
- **high (重要)**: 需要关注和处理的重要告警，可能影响系统性能
- **medium (一般)**: 需要注意的一般告警，不会立即影响系统运行
- **low (次要)**: 提示性告警，用于早期预警

### 3.2 告警状态

- **Active (活动)**: 新产生或尚未确认的告警
- **Acknowledged (已确认)**: 用户已经知晓并确认的告警
- **Resolved (已解决)**: 告警条件已消除
