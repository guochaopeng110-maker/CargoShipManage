# 能力：健康评估 (health-assessment)

## 新增要求 (ADDED Requirements)

### 要求：动态设备健康仪表盘
系统必须动态显示从设备管理模块检索到的所有设备的健康评估卡片。

#### 场景：显示多个系统卡片
- **当 (WHEN)** 健康评估页面加载时
- **并且 (AND)** 系统从 API 检索到 8 个设备的列表时
- **那么 (THEN)** 仪表盘上应当渲染 8 个独立的健康评估卡片。

### 要求：纯历史健康评估
系统不得在健康评估页面上使用实时 WebSocket 数据来生成健康分数或状态指示器。所有数据必须通过 HTTP API 从历史报告中派生。

#### 场景：在没有 WebSocket 的情况下加载数据
- **当 (WHEN)** 用户导航到健康评估页面时
- **那么 (THEN)** 系统应当通过 HTTP GET 请求获取最近的健康报告
- **并且 (AND)** 不得显示任何“实时”连接警告或状态图标。

### 要求：增强型报告历史筛选
系统应当允许用户按设备 ID、设备名称和日期范围筛选健康报告历史。

#### 场景：按设备和日期筛选
- **当 (WHEN)** 用户从筛选下拉菜单中选择特定设备时
- **并且 (AND)** 设置了日期范围（例如过去 7 天）时
- **那么 (THEN)** 健康报告表格应当刷新，仅显示符合这些条件的报告。
